vManage API Calls for SDWAN

This Portal was created to demonstrate the power of Cisco SD-WAN API.vManage API vManage API is a powerful tool that we can use to sell the value of our SD-WAN solution. Is for that reason that during some PoVs running in a local test bed, some customers as financial entities requested to see some specific use case for their environment in hybrid WAN scenarios (MPLS + INET).

Main use cases

Their main concern was related with the fact that use of INET in any flavor (4G, ADSL, GPON, etc) , SD-WAM is programed to allow Dynamic Public IP address as a source of IPSEC tunnels to build the Fabric Overlay. In any case that a "bad guy" has stolen Branch vEdge/cEdge, then from anywhere with INET access, it could get access to the corporate network until the SD-WAN administrator knows that vEdge/cEdge has been stolen.

Other use cases as QoS manipulation are created to interact with vManage in order to create and modify the QoS behavior in massive remote sites
